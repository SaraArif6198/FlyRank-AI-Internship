## 5 Target Tasks and Success Definitions for Subsequent Modules
These three tasks are selected for deeper AI-integration testing in subsequent assignments.

### Task 1: Automating SQL Migration Scripts for DG Cement Work Tracker
*   **Classification:** `Collaborate with AI`
*   **Context:** Writing and running migration files to introduce new audit log tables and database constraints.
*   **Measurable Success Criteria:**
    *   Schema changes run without syntax errors.
    *   All existing production rows are preserved with no data loss.
    *   Execution completes in less than 500 milliseconds on a 10000 row database.

### Task 2: Implementing Compiler Optimization Passes in SQL2Pandas
*   **Classification:** `Just Me`
*   **Context:** Writing AST manipulation logic for dead-code elimination and projection pruning in the transpiler.
*   **Measurable Success Criteria:**
    *   The transpiler passes all unit tests.
    *   Simplifies a 5 JOIN SQL query AST by removing at least 2 redundant nodes.
    *   No regression bugs are introduced in the generated Pandas queries.

### Task 3: Engineering the LangGraph StateGraph Node logic in DataSage
*   **Classification:** `Collaborate with AI`
*   **Context:** Setting up linear pipeline nodes (analysis, narrative, visualization) and ensuring correct JSON state transitions.
*   **Measurable Success Criteria:**
    *   The entire pipeline executes in less than 15 seconds.
    *   Gracefully handles empty or corrupted CSV files by returning structured error states without crashing.
    *   Output JSON matches the frontend Pydantic schema structure.
